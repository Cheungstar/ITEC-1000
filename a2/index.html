The architecture of computers has not changed at all throughout time. The new concept that John Von Neumann and Alan Turing came up with between 1945 to 1951 is the concept of the “stored-program” architecture, where not only should the data be stored in memory, but that program processing that data should also be stored in the same memory. 4.      Question: Why are there two different registers (MAR and MDR) associated with memory? What are the equivalents in the Little Man Computer? Explain clearly and coherently.

 

 

MAR is the memory address location and only holds the location address but not the contents of what that memory location contains. A decoder is connected to the MAR, which will interpret or decode the memory address and then activates a single address line into the memory. Each group of cells in the memory has a separate address line. Therefore, if there are n bits of addressing, there will be 2n address lines.

MDR is the memory data register, with the purpose of holding a data value that is stored to or retrieved from the memory location the memory address register (MAR) is currently addressed to. The memory data register is designed in order to be effectively connected to every cell in the memory. Each bit of the MDR is connected in a column to the corresponding bit of every location in memory. However, the addressing method assures that only a single row of cells is activated at any given time and so only one memory location will be addressed at any one time.

5.      Question: As computer words get larger and larger, there is a law of diminishing returns; the speed of execution of real application programs does not increase and may, in fact, decrease. Why do you suppose that this is so?

 

 

The law of diminishing returns is a phenomenon that happens in all productive processes, and states that adding more of only one factor of production while holding all other factors constant will begin to result in lower incremental per-unit of returns at some point. So as computer words get larger, then there needs to be enough op code bits to support the instruction set but also there must be enough address field bits to be able to cope with the increase of memory that can be addressed. Therefore, the speed of execution of real application programs will not increase further and may actually decrease.
